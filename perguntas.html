a) O atributo placeholder na tag input com type="text" é usado para fornecer um texto de exemplo ou uma dica dentro do campo de entrada. Este texto aparece no campo antes do usuário começar a digitar e desaparece quando o usuário insere algum texto.

b) O atributo checked na tag input com type="radio" ou type="checkbox" é usado para definir o estado inicial do elemento como selecionado ou marcado

c) O atributo value na tag input é usado para definir o valor inicial do campo de entrada. O comportamento deste atributo varia dependendo do tipo do input.

d) para agrupar vários botões de rádio em um formulário de modo que apenas um deles possa ser marcado de cada vez, você deve usar o atributo name. Todos os botões de rádio que compartilham o mesmo valor de name fazem parte do mesmo grupo e, portanto, apenas um deles pode ser selecionado de cada vez

e) A tag label é usada para definir rótulos para elementos de formulário. Ela melhora a acessibilidade e a usabilidade do formulário, pois permite que os usuários cliquem no rótulo para focar ou selecionar o campo de entrada associado. O atributo for na tag label é utilizado para associar o rótulo a um elemento de formulário específico. O valor do atributo for deve ser igual ao valor do atributo id do elemento de formulário ao qual o rótulo se refere. Isso cria uma ligação entre o rótulo e o elemento de formulário, permitindo que o rótulo seja clicável e automaticamente foque ou selecione o elemento correspondente.

f) O atributo selected na tag option é usado para definir a opção inicial que será selecionada em um menu suspenso select quando a página é carregada. Se o atributo selected estiver presente em uma tag option essa opção será mostrada como a opção padrão selecionada.

g) O atributo required nas tags de formulário é usado para indicar que o campo de entrada deve ser preenchido antes que o formulário possa ser submetido. Se um campo marcado como required estiver vazio, o navegador impedirá o envio do formulário e exibirá uma mensagem de erro, solicitando ao usuário que preencha o campo obrigatório.

h) A tag textarea é usada para criar um campo de entrada de texto multilinha em um formulário. Diferente de um input type="text", que é um campo de entrada de linha única, o textarea permite que os usuários insiram grandes quantidades de texto, como comentários, descrições, ou outras entradas de texto mais longas

i) A tag textarea é usada para criar um campo de entrada de texto multilinha em um formulário. Diferente de um input type="text", que é um campo de entrada de linha única, o textarea permite que os usuários insiram grandes quantidades de texto, como comentários, descrições, ou outras entradas de texto mais longas.

j) GET: Envia os dados do formulário anexados à URL como uma string de consulta (query string). Os dados do formulário são visíveis na barra de endereços do navegador. Geralmente usado para solicitações que não alteram o estado do servidor, como consultas de pesquisa. Limitação de tamanho de dados, pois os navegadores e servidores podem impor limites ao comprimento da URL. POST: Envia os dados do formulário no corpo da solicitação HTTP. Os dados não são visíveis na barra de endereços do navegador. Geralmente usado para solicitações que alteram o estado do servidor, como envio de dados de um formulário de cadastro ou login. Não há limitações práticas no tamanho dos dados enviados Segurança: POST é mais seguro que GET para enviar dados sensíveis porque os dados não são expostos na URL. Cache: Navegadores podem armazenar em cache solicitações GET, mas não POST. Idempotência: Solicitações GET são idempotentes (não devem alterar o estado do servidor), enquanto POST pode alterar o estado no servidor.

k) Para evitar que o valor digitado em campos sensíveis, como senhas, apareça na barra de endereços do navegador, você deve usar o atributo method com o valor "post" na tag
. Isso garante que os dados do formulário sejam enviados no corpo da requisição HTTP, em vez de serem anexados à URL como uma string de consulta (GET). Dessa forma, os dados sensíveis não serão visíveis na barra de endereços do navegador.

l) Para formulários que serão usados para fazer upload de arquivos, o valor do atributo method na tag form deve ser "post". Isso ocorre porque o método HTTP POST permite enviar dados mais complexos e grandes, como arquivos binários (como imagens, documentos, etc.), através do corpo da requisição HTTP.

m) As tags fieldset e legend são usadas em conjunto para criar um grupo de controles de formulário relacionados, como campos de entrada, caixas de seleção, botões de rádio, etc. Elas são úteis para organizar e estruturar formulários, especialmente quando há várias seções ou grupos de campos relacionados. fieldset: Esta tag é usada para agrupar os controles de formulário relacionados em uma seção visualmente destacada. Ela cria uma caixa ou um contorno ao redor dos controles agrupados, ajudando a melhorar a acessibilidade e a usabilidade do formulário. legend: Esta tag é usada dentro de fieldset e serve para fornecer um rótulo ou uma legenda para o grupo de controles dentro do fieldset. A legenda geralmente descreve brevemente o propósito ou o contexto dos controles dentro do grupo.
